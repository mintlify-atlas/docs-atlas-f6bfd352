ProgressiveBlurHeader is a drop-in SwiftUI component that gives your app a sticky header with a progressive blur effect — the same style used by Apple Music, Photos, and the App Store. Content scrolls freely underneath the header, becoming increasingly blurred and tinted as it passes through, without ever being clipped or hidden.

## The problem it solves

Building a progressive blur header natively in SwiftUI is surprisingly difficult. SwiftUI has no built-in API for variable-radius blur, and common workarounds each come with painful trade-offs:

- **Clipped content** — using `.clipped()` or a `ZStack` with a solid header background hides scrolling content instead of blurring it
- **Flat uniform blur** — applying a fixed `.blur()` modifier produces a single-radius effect that looks nothing like Apple's treatment
- **Brittle `UIViewRepresentable` wrappers** — manual UIKit bridges tend to break with SwiftUI's layout system, especially during safe area changes or size class transitions

AI coding assistants consistently struggle with this pattern, often requiring many iterations to produce a working result. ProgressiveBlurHeader gives you a correct, App Store–approved solution with a single view.

## How it works

The component uses a three-layer `ZStack` where each layer has a dedicated role:

<CardGroup cols={3}>
  <Card title="Back layer: ScrollView" icon="layer-group">
    Your scrollable content sits in a `ScrollView` that is never clipped. Content flows freely behind the header at full fidelity.
  </Card>
  <Card title="Middle layer: blur + tint" icon="wand-magic-sparkles">
    A `VariableBlurView` sits above the scroll content. It applies a progressive blur — strongest at the top, fading to zero below the header — with an adaptive gradient tint overlay.
  </Card>
  <Card title="Front layer: your header" icon="rectangle-history">
    Your header view floats on top, with no opaque background. The `VariableBlurView` beneath it acts as the visual background, letting the blur show through.
  </Card>
</CardGroup>

The blur engine is powered by [VariableBlur](https://github.com/nikstar/VariableBlur) by nikstar, which uses the same obfuscated private API that Apple uses internally. It has 495+ stars on GitHub and is App Store approved.

## Key features

- Single view (`StickyBlurHeader`) — wrap your header and content, done
- Progressive blur that fades from maximum intensity at the top to clear below the header
- Adaptive tint that automatically switches between a white overlay (light mode) and black overlay (dark mode)
- Header height measured automatically via `GeometryReader` and `PreferenceKey` — no hardcoded values needed
- Four tunable parameters for blur intensity, fade length, and tint strength
- No `UIViewRepresentable`, no clipping, no layout hacks

---
title: "Quickstart: add StickyBlurHeader to your SwiftUI app"
description: "Add a native-quality progressive blur sticky header to your SwiftUI app in under five minutes, from package install to working UI."
---

ProgressiveBlurHeader is designed to get you from zero to a working progressive blur header as fast as possible. The entire integration is a single `StickyBlurHeader` view — you provide two `@ViewBuilder` closures and it handles the rest. This guide walks you through the complete setup.

<Steps>
  <Step title="Add the package">
    Open your project in Xcode and go to **File → Add Package Dependencies**. Paste the repository URL:

    ```
    https://github.com/dominikmartn/ProgressiveBlurHeader
    ```

    Select **Branch: main** and click **Add Package**. Xcode will resolve the package and its `VariableBlur` dependency automatically.

    <Info>
      Prefer `Package.swift`? See the [Installation](/installation) page for the dependency declaration.
    </Info>
  </Step>

  <Step title="Import the module">
    At the top of the Swift file where you want the header, add the import:

    ```swift
    import ProgressiveBlurHeader
    ```
  </Step>

  <Step title="Wrap your views in StickyBlurHeader">
    Replace your existing `ScrollView` or top-level view with `StickyBlurHeader`. Pass your navigation bar content as the `header` closure and your list or scroll content as the `content` closure:

    ```swift
    import SwiftUI
    import ProgressiveBlurHeader

    struct MyView: View {
        var body: some View {
            StickyBlurHeader {
                // Your header — NO opaque background!
                HStack {
                    Button("Back") { }
                    Spacer()
                    Text("Title").font(.headline)
                    Spacer()
                    Button("Settings") { }
                }
                .padding()
            } content: {
                // Your scrollable content
                ForEach(0..<30) { i in
                    Text("Item \(i)")
                        .padding(.horizontal, 20)
                        .padding(.vertical, 12)
                    Divider()
                }
            }
            .background(Color(.systemBackground))
        }
    }
    ```

    <Warning>
      The header closure must **not** have an opaque background (e.g. `.background(Color.blue)`). The `VariableBlurView` in the middle layer is the background. Adding an opaque background to the header hides the blur effect entirely.
    </Warning>
  </Step>

  <Step title="Set a background on the parent view">
    Apply `.background()` on the `StickyBlurHeader` itself (not on the header closure). This sets the canvas color that shows through the blur. For most apps, `Color(.systemBackground)` is the right choice — it adapts automatically to light and dark mode.

    ```swift
    StickyBlurHeader { ... } content: { ... }
        .background(Color(.systemBackground))
    ```
  </Step>
</Steps>

## Result

Your content now scrolls freely underneath the sticky header. As items pass behind it they become increasingly blurred and tinted — the blur is strongest at the top (behind the Dynamic Island and status bar) and fades to clear just below the header edge.
